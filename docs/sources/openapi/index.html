OpenAPI sync

If your app serves an OpenAPI document, this is the source to use. It gives the cleanest tool names and the most accurate types.

What you need

  • A running HTTP server that serves an OpenAPI 2 or 3 document, either at a URL like /openapi.json or as a file on disk.
  • appctl installed (cargo install appctl).

The demo in this repo

examples/demos/openapi-fastapi/ is a FastAPI app with one endpoint. FastAPI generates the OpenAPI document automatically at /openapi.json.

Set up Python

You need Python 3.11 or newer.

cd examples/demos/openapi-fastapi
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Start the server

uvicorn main:app --host 127.0.0.1 --port 8000

Confirm it is up:

$ curl -s http://127.0.0.1:8000/openapi.json | head -c 80
{"openapi":"3.1.0","info":{"title":"appctl demo API","version":"1.0.0"},"paths"

Sync appctl

In another terminal:

appctl sync --openapi http://127.0.0.1:8000/openapi.json \
  --base-url http://127.0.0.1:8000 --force

Real output on this machine with appctl 0.2.0:

Synced Openapi: 1 resources, 1 tools written to .appctl

Look at what was generated:

$ cat .appctl/schema.json | python3 -c "import json,sys; s=json.load(sys.stdin); [print(r['name'], [a['name'] for a in r['actions']]) for r in s['resources']]"
widgets ['create_widget_widgets_post']

Call it through chat

This step needs a language model configured. See install for how to set a provider in .appctl/config.toml. 

appctl chat "create a widget named Demo"

What appctl does in this mode

  • Fetches the OpenAPI document (or reads it from disk).
  • For each path and operation, creates one tool with JSON Schema parameters.
  • Marks the tools as provenance=declared. This is the highest trust level because the server itself said these routes exist.
  • Reads securitySchemes from the spec and picks the matching AuthStrategy.

Verify the tools are reachable

$ appctl doctor
tool                             method path         HTTP  verdict
create_widget_widgets_post       POST   /widgets      200  reachable

Pass --write to mark reachable routes as provenance=verified in the schema, so the agent knows they have been exercised against the live server.

When OpenAPI is not a good fit

  • Incomplete specs produce incomplete tools. Lint your spec with Spectral before syncing.
  • If the spec is served behind login, save it to a file and pass the file path instead of the URL.
  • Webhook and callback operations are ignored.
